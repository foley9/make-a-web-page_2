HTML - Hyper Text Markup Language HTML is the standardized system for formatting the text documents that make up web pages. Most of HTML is regular text, but then there are tags and elements. Tags tell the browser how to display the elements of the web page. From formatting the text, like bold or italic, to adding images, videos and hyper links.

Tags
Tags are instructions to the browser on how to display the data in the html document. There are many tags, tags to bold text, italicize, center, line break.
Inline vs Block Elements
HTML elements are either inline, just text entered like in a notepad or word doc, or Block, which puts the element into a small invisible box.

Structured Code
Structured programming is a style of code that focuses on being easy to read and update through the use of sub-routines and for and while loops; and avoidance of non-structured commands like the go-to command. Structured code usually has 1 entry point that starts at the top and works down to one entry point at the bottom. Older non-structured programming languages, like Cobal, used Go-to commands which would have you jump all over from line to line and it made it very hard to read and to maintain.
Structured code is easy to look at and understand what is happening as related elements are palced under each other in a tree-like format. Like a family tree, the parent would be listed, then branch to the children, tree-like coding has related elements branch under their parent element.
DOM Document Object Model - DOM is a logical structure of related elements being indented in a tree-like model. Since everything is a box, this basically means a box inside of another box is indented under the larger box in the document code. DOM is the standard convention for representing and interacting with elements in html.

Lesson 3: Adding CSS Style to HTML structure

CSS: Cascading Style Sheets
CSS is a seperate language that is used to create the style of a webpage.HTML documents are instructed to reference a CSS document to look up the styles to use.
CSS uses a Class to define style, things like font, size, spacing, and color can all be defined in a class. Then elements in the html document are assigned to that class and all the class properties will be applied to that element.
Avoid Repetition CSS allows you to set style classes, then you can apply that class to any element and not have to repeat all of the tags that make up that style. This allows you to not have to reapply the tags over and over again to mulitple elements.
Consistency Appying css styles to elements is a good way to keep a consistant look throughout your website. And if you want to make a change, changing the one class will update all elements assigned to that class.
Debugging HTML and CSS
steps for finding errors in html and css documents:
first check syntax, check all element tags are closed, look for missing characters like semicolons, quotes or brackets
use a validation tool that scans your code and shows any errors. W3C Validator W3C is an example of an html validation tool.
Cross-browser issues a common problem is a website displaying different on different web broswers. to minimize these errors, use CSS reset, like "normalize" this makes most browsers perform in a more standardized way.
use Developer tools in chrome or firefox to turn on and off elements and see how they effect the page. this is a good way to see which element may be causing the display issue.
